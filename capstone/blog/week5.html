Week5 - February 26, 2021

Journal Entry 5

For this week’s progress, I work on usability test and redesign. I created a usability test script and use my current wireframe to conduct usability test, and modify the design based on users suggestions.

Usability Test Script

I wrote a usability test script and performed 8 usability tests using the wireframe I designed, use this link to visit the usability script.

In the usability test, I will first ask the user few questions about the internet and apps.

  1. How often do you use the internet? And how many hours per day/week?
  2. Do you usually visit webpages on PC/MAC or smartphone?
  3. Which kind of website do you usually visit?
  4. Have you ever used any AR app?
  5. What’s your favorite AR app?
  6. What’s your AR app experience?

Next, I will let the user start the Home Page Tour and ask the user:

  1. Which do you think this app is?
  2. What can you do here?
  3. Which information can you find here?

I will give the user 3-5 minutes for this process.


Then I will let the user complete the following 6 tasks:

  1. You want to see landmarks from Japan.
  2. You want to see the Great Buddha Hall in AR.
  3. You want to take a picture of the Great Buddha Hall.
  4. You want to turn off the music.
  5. You want to see the Forbidden City in full scale AR.
  6. You want to read the description of the Forbidden City.

Suggestions

After the usability test, most of the participants complete thoes tasks, and I got following suggestions:

  1. Make the model accurate enough.
  2. Add storytelling function.
  3. Listen to the sound effect when people visit landmarks through AR.
  4. Click on some parts of the building and play the voice introduction.
  5. View the building directly on the list page, skip the building introduction page.
  6. Zoom in and zoom out the building by hand, and remove the Full scale AR button.

The user thinks why two buttons are needed if you can zoom in and out the model by hand? "Full scale AR" is a blue button, so users are likely to click on the "Full scale AR" button, but after clicking the "Full scale AR" button, a warning page will pop up, telling the user that it can only be used in a large space, but many users use it indoors , so this feature cannot be used. So this model is likely to be skipped. The user suggests that all models have only one button "View in AR", and allowing users to zoom in and out of the building according to their own space.

Redesign

After the usability test, I found that many users have no experience in using AR apps. I need to make it clearer that they need to scan the ground before they can use a AR app. I added a page to ask the user to scanning the ground and added a A dynamic icon to guide the user to scan the surface and place the model on the ground.
I removed the "Full Scale AR" button and set the buttons of all models to "view in AR" and allowed the user to manually zoom in and out of the model in AR mode.
Since the list view may require users to rescan the ground every time they exit from a model, I refer to my design precendent: Civilisations AR, which allows users to place a globe on the ground first, and then select a location from the globe, after selecting a landmark, click "view in AR" to replace the earth model with a landmark model, which avoids the user from repeatedly scanning the ground and improves the interaction efficiency.
In order to improve the efficiency of interaction, I also removed the building information page, allowing users to directly select a building from the earth page and click "view in AR" to view the landmark building.
In addition, I also changed the black mask of the information page to a frosted glass material.
